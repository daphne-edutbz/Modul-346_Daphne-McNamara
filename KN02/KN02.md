## C) Zugriff mit SSH-Key (40%)

Für den Zugriff auf die EC2-Instanz wird SSH mit Public/Private-Key-Verfahren verwendet.
Auf der Instanz liegt nur der öffentliche Schlüssel (.pub), während der private Schlüssel (.pem) ausschliesslich auf meinem lokalen Computer gespeichert ist.

**SSH-Zugang mit dem richtigen Schlüssel (Daphne1)**

```bash
ssh ubuntu@18.209.67.28 -i C:\Projects\GitHub\Modul-346_Daphne-McNamara\KN02\Daphne1.pem -o ServerAliveInterval=30
```

## Leitfragen / Checkpoints

1. Ich komme mit der Schulungsumgebung von AWS zurecht und kenne die Struktur der Module ✅


2. Im Learner Lab kann ich Umgebungen starten und stoppen, Budget einsehen und weiss wie lange meine Umgebung bestehen bleibt. ✅


3. Ich weiss was EC2 bedeutet und kenne die Inhalte, die ich dort erstellen und verwalten kann. ✅
- EC2 bedeutet Elastic Compute Cloud und stellt virtuelle Server (Instanzen) zur Verfügung.
- "Virtual Servers in the cloud"

  
4. Ich kann eine Instanz/virtuellen Server erstellen und kann die Einstellung zum Speicher, RAM, CPU bestimmen und einstellen. ✅


5. Ich weiss wie die private/public Key Verschlüsselung funktioniert und kann ausführen wo (Gerät und Verzeichnis) der Public und der Private Key liegen muss. ✅
- private Schlüssel (.pem) auf meinem lokalen Gerät liegen muss und niemals hochgeladen werden darf.
- öffentliche Schlüssel (.pub) auf dem Server gespeichert ist (bei EC2 automatisch in ~/.ssh/authorized_keys).
- Login funktioniert nur, wenn der private Schlüssel zu einem der öffentlichen Schlüssel passt.

6. Ich kann den öffentlichen Schlüssel aus dem privaten Schlüssel extrahieren. ✅
- mit den befehl `ssh-keygen -y -f <privatekey.pem>`
  
7. Ich kenne den Befehl ssh und die verschiedenen Parameter und kann mich damit auf einen Remote Server einloggen. ✅
- `-i <key.pem>` – gibt den privaten Schlüssel an
- `-o ServerAliveInterval=30` – verhindert Timeout
- `user@ip` – Zieladresse

Beispiel:
``Bash
ssh ubuntu@<öffentliche-ip> -i Daphne1.pem -o ServerAliveInterval=30
``
